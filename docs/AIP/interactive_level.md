# Application/Web Interactive Levels

## **1. Static**

* **Definition**: Content is pre-rendered and served as-is from the server.
* **User Interaction**: Minimal to none.
* **Technology Examples**:

  * HTML/CSS files hosted on a server.
  * No client-side JavaScript logic.
* **Use Cases**:

  * Marketing websites
  * Documentation
* **Pros**: Fast loading, easy to deploy, secure.
* **Cons**: No personalization or interactivity without page reloads.

---

## **2. Dynamic**

* **Definition**: Content is generated on-the-fly based on user requests or data.
* **User Interaction**: Moderate to high; content may change based on user inputs or sessions.
* **Technology Examples**:

  * PHP, Django, Node.js with server-side rendering.
  * Database-driven websites.
* **Use Cases**:

  * News websites, e-commerce platforms.
* **Pros**: Personalized content, data-driven.
* **Cons**: Heavier load on the server, slower than static sites.

---

## **3. Low Interactive Level**

* **Definition**: Pages include some basic interactive features.
* **User Interaction**: Limited to simple UI components.
* **Technology Examples**:

  * HTML + minimal JavaScript/jQuery.
  * Forms, basic buttons, tooltips.
* **Use Cases**:

  * Feedback forms, simple dashboards.
* **Pros**: Easy to develop, lightweight.
* **Cons**: Limited user engagement or dynamic feedback.

---

## **4. High Interactive Level**

* **Definition**: Full client-side interactivity with rich, responsive user experience.
* **User Interaction**: Real-time, seamless, SPA-like (Single Page Application).
* **Technology Examples**:

  * React, Vue, Angular, WebSockets, GraphQL.
  * Live chat, drag-and-drop, dashboards with real-time updates.
* **Use Cases**:

  * Web apps, collaborative tools, social media platforms.
* **Pros**: Engaging UX, dynamic data updates.
* **Cons**: Higher complexity, more client-side resources required, SEO may need special handling.
